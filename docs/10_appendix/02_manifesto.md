# The DHARMA Manifesto

## 1. Sovereignty is the Default
Data should belong to the people who create it. Software should be a tool for people to coordinate, not a trap for their information. DHARMA is built to empower individuals and organizations to own their history without asking for permission.

## 2. Shared Truth is Possible
Distrust is expensive. We build systems of verification so that we can afford to trust. Shared truth is not about having a central authority; it is about having a shared law that is impossible to break without leaving a signature.

## 3. History is the State
The present is a fleeting moment. The past is the only foundation we have for the future. By preserving history, we preserve context, accountability, and the ability to learn from our mistakes.

## 4. Logic must be Literate
If a law is written in code that only a machine can read, it is not a law; it is a mechanism. DHARMA DHL is designed to be readable by humans and executable by machines. The documentation IS the contract.

## 5. Simplicity is Reliability
A system that is too complex to understand is impossible to secure. DHARMA strives for a minimal kernel, a small binary size, and a deterministic execution environment. Reliability is the result of removing moving parts.

**We are building the "Emacs of the 2030s" - a universal, programmable, sovereign environment for the digital life of humanity.**
