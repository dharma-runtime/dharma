# Audit & Forensics

DHARMA treats auditability as a first-class property.

---

## 1) Immutable History

All accepted assertions remain in the subject log. Truth is always reconstructible by replay.

---

## 2) REPL Audit Tools

- `tail <n>`: recent assertions
- `show <id>`: raw assertion
- `why <path>`: explain a state value
- `prove <id>`: full validation report

---

## 3) Provenance in Queries

DHARMA-Q results include assertion IDs and subject IDs so every row can be traced back to source truth.

---

## 4) Planned

- Signed receipts
- Formal proofs
- Cross-subject provenance links
